## กรณีศึกษาที่ 1 : ArrayCaseStudy

### ผลลัพธ์

```
===== ผลการวิเคราะห์คะแนน =====
คะแนนรวม = 62
คะแนนเฉลี่ย = 6.2
คะแนนสูงสุด = 10
คะแนนต่ำสุด = 2
จำนวนนักศึกษาที่ได้คะแนนตั้งแต่ 7 คะแนนขึ้นไป = 5

นักศึกษาที่ควรได้รับการทบทวน
นักศึกษาคนที่ 3 ได้ 4 คะแนน
นักศึกษาคนที่ 8 ได้ 3 คะแนน
นักศึกษาคนที่ 10 ได้ 2 คะแนน

Time Complexity : O(n)
```

## กรณีศึกษาที่ 2 : StackCaseStudy

### ผลลัพธ์

```
===== ข้อมูลใน StackCaseStudy =====
[พิมพ์ Data, พิมพ์ Structure, ลบ Structure, พิมพ์ Algorithm, พิมพ์ Java]

ทำการ Undo 2 ครั้ง
Undo : พิมพ์ Java
Undo : พิมพ์ Algorithm

Stack หลังจาก Undo
[พิมพ์ Data, พิมพ์ Structure, ลบ Structure]

หลักการทำงานของ Stack
LIFO (Last In First Out)
ข้อมูลที่เข้ามาล่าสุด จะถูกนำออกก่อน

Time Complexity
push() = O(1)
pop() = O(1)
peek() = O(1)
```

## กรณีศึกษาที่ 3 : QueueCaseStudy

### ผลลัพธ์

```
===== คิวผู้ป่วยเริ่มต้น =====
[P001, P002, P003, P004, P005]

เรียกผู้ป่วยเข้ารับบริการ
รับบริการ : P001
รับบริการ : P002

ผู้ป่วยคนถัดไป : P003
จำนวนผู้ป่วยที่รอ : 5

สถานะของ Queue
[P003, P004, P005, P006, P007]

หลักการทำงานของ Queue
FIFO (First In First Out)
ผู้ที่เข้าคิวก่อน จะได้รับบริการก่อน

Time Complexity
add() = O(1)
remove() = O(1)
peek() = O(1)
```
